퀵 정렬

저번시간에 공부한 버블정렬 이외에도 가장 빠른 정렬 알고리즘 중의 하나로 널리 사용되고 있는 퀵정렬에 대해서 개념 및 코드 구현을 해보았습니다.
퀵 정렬이라는 이름은 이 알고리즘의 정렬 속도가 매우 빠른 데서 착안해 찰스 엔터니 리처드 호어가 붙인 이름입니다.

퀵 정렬 이미지

처음에 그림만보면 도대체 어떻게 정렬하는지 가늠되지가 않았습니다. 퀵 정렬에 대해서 공부하기전에 사전에 알아야하는 단어가 있는데 Pivot(피벗)이라고 하는 녀석을 알아야 합니다. 피벗은 어떤 리스트 안의 값들 중에 임의로 하나를 기준으로 잡아서 그 기준보다 작은 그룹과 큰 그룹으로 정렬을 수행 합니다.

위의 그림에서는 피벗을 5로 잡았습니다. 5를 기준으로 왼쪽에는 5보다 작은 그룹이, 오른쪽에는 5보다 큰 그룹으로 정렬되어 있습니다. 이렇게 퀵 정렬은 하나의 리스트를 피벗을 기준으로 두 개의 비균등한 크기로 분할하고 분할된 부분 리스트를 정렬한 다음, 두 개의 정렬된 부분 리스트를 합하여 전체가 정렬된 리스트가 되게 하는 방법입니다.

이러한 퀵 정렬을 Java 소스코드로 구현해보았습니다.

퀵 정렬 소스코드

위의 코드와 실행과정을 보면 먼저 배열을 두 그룹으로 나누는 순서에 대해서 살펴보겠습니다. 위의 배열에서 값을 입력 받아서 퀵 정렬로 오름차순을 수행하는 과정입니다. 피벗을 x에, 왼쪽 끝 요소의 인덱스 값을 pl, 오른쪽 끝 요소의 인덱스를 pr에 넣었습니다. 그룹을 나누기 위해 피벗 이하의 요소를 배열 왼쪽으로, 이상의 요소를 배열 오른쪽으로 옮겨야 합니다.

먼저 퀵 정렬을 수행하기 위해서 quickSort 메서드의 인자로 배열 x, 배열의 첫번째 인덱스인 0, 배열의 맨 오른쪽 끝 인덱스를 넣어서 호출을 합니다. 그리고 while문으로 배열 왼쪽에 있는 pl 값이 피벗보다 큰 값을 찾을 때까지 오른쪽으로 계속 스캔을합니다.
a[pl] >= x이면 pl 값은 증가하지 않습니다. 마찬가지로 배열 오른쪽에 있는 pr도 피벗보다 작은 값을 만나기전까지 while문으로 계속 왼쪽으로 스캔하게 되고, a[pr] <= x이면 pr값도 감소하지 않습니다.
pl <= pr이 성립되면 pl과 pr이 가리키는 요소(피벗보다 작은 값과 큰 값)위치를 교환하는 swap 메서드를 수행하게 됩니다.
이렇게 계속 수행을 하면 a[0] ~ a[4]와 a[5] ~ a[8]으로 나누어집니다. 그러면 이 두 그룹을 다시 같은 방법으로 나눕니다. 요소의 개수가 1개
인 그룹은 더 이상 그룹을 나눌 필요가 없으므로 요소의 개수가 2개인 이상인 그룹만 나누면 됩니다.

pr이 a[0]보다 오른쪽에 있으면(left < pr)왼쪽 그룹을 나눕니다.

pl이 a[8]보다 왼쪽에 있으면(pl < right)오른쪽 그룹을 나눕니다.

left < pr, pl < right는 모두 그룹의 개수가 1개인 경우에는 성립하지 않는 조건입니다. 다시 말해 요소의 개수가 2개 이상인 그룹만이 나누기 위해 필요한 조건입니다.
퀵 정렬은 분할 정복 알고리즘이므로 재귀 호출을 사용하여 구현할 수 있습니다.
