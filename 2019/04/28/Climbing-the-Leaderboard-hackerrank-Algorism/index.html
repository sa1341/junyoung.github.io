HackerRank

요즘은 배민이나 카카오 같은 대기업에서도 해커랭크를 이용하여 문제를 출제하기 때문에 일주일에 한번씩 머리를 굴릴겸 한 문제씩 시간이 오래걸리더라도 풀려고 노력하고 있습니다.
게다가 문제들이 영어로 되어 있기 때문에 영어실력도 늘릴겸 자주 애용하는 사이트입니다.
계속 easy 모드만 풀다가 medium 문제를 풀어봤는데… 생각보다 많이 어려웠고 시간도 오래걸려서 다른분의 블로그를 참고하여 좋은 풀이방법에 대해서 공부할 수 있었습니다.
이러한 유형의 문제가 출제될때마다 접근하는데 시간을 줄일 수 있게 효율적으로 풀이하는 방법에 대해서 리뷰하겠습니다.

Climbing the Leaderboard

먼저 문제를 보자면 아래와 같습니다.

Alice is playing an arcade game and wants to climb to the top of the leaderboard and wants to track her ranking. The game uses Dense Ranking, so its leaderboard works like this:

해석하는데도 꽤 시간이 걸렸습니다… 이럴줄 알았으면 영어공부도 많이 해놓는건데…항상 후회만 ㅎㅎ… 앨리스는 아케이드 게임을 하고 있고 리더보드(랭킹이 나와있는 게시판)의 최상위 순위에 랭크 되기를 원하고 있습니다. 그리고 그녀의 랭킹을 추적하기를 원하죠.
그 게임은 Dense 랭킹을 사용한다는데…Dense 랭킹이 뭔지는 모르겠습니다. 문제를 해결하는데 그렇게 핵심키워드는 아닌거 같아서 대충 이런게 있구나 하고 건너뛰었습니다. 여기까지는 서론이고 이제 리더보드의 규칙성에 대해서 살펴보겠습니다.

  • 가장높은 점수를 가진 플레이어는 리더보드의 1등으로 랭크가 되어진다.
  • 같은점수를 받은 플레이어들은 같은 랭킹 숫자를 부여 받고, 다음 랭킹의 플레이어는 즉시 다음 랭킹번호를 부여 받는다.

대충 이렇게 리더보드 시스템이 구성되어 있습니다.
이제 본격적으로 예시를 보면 100, 90, 90, 80점을 가진 4명의 플레이어가 존재한다면, 이 플레이어들의 각각의 랭크는 1, 2, 2, 3등 입니다.

만약 앨리스의 점수가 70, 80, 105 점을 받는다면 각각의 게임을 한 후에 그녀의 랭킹은 4등, 3등, 1등이 되는겁니다. 확실히 예시를 보니까 더욱 이해가 빨라졌습니다. 그전에는 해석하느라 무슨말인지도 기억이 안나더군요…

한마디로 리더보드라는 랭크 게시판에서 앨리스의 등수들을 리턴하는게 해당 문제의 핵심입니다.

Input Format도 4개나 되는군요…
첫번째 input parameter는 리더보드의 랭크되어 있는 플에이어들의 수를 의미합니다.
두번째는 리더보드에 내림차순으로 정렬되어 있는 플레이어들의 점수들을 가진 배열 변수입니다.
세번째는 앨리스의 플레이 횟수를 나타내는 정수를 의미합니디.
네번째는 앨리스의 게임 횟수동안 기록한 점수들을 가진 배열을 의미합니다.

이 문제는 뭔가 쉬우면서도 복잡해 보였지만 그래도 친절하게 그림과 표를 통해서 자세하게 예시를 보여주고 있어서 문제자체를 이해하고 어떤 output을 구하는지 많은시간이 걸리지는 않았습니다.

그림은 아래 이미지와 같습니다.

각 파라미터가 의미하는 값들의 예시를 그림으로 보여주고 있습니다.
1.리더보드에 랭크되어 있는 플레이어의 수
2.각각의 랭크되어 있는 플레이어들의 점수를 가진 배열
3.앨리스의 플레이어 횟수
4.앨리스가 플레이어 한 후에 각각의 점수들을 가진 배열

위의 그림만 보셔도 대충 어떤 output을 리턴해야하는지 알 수 있을겁니다.

이제 어떻게 앨리스의 랭킹 등수를 출력하는지 자바로 구현해보았습니다.

먼저 리더보드에 등록되어 있는 플레이어의 등수와 점수를 어떻게 관리할까 고민을 하였습니다. 처음에는 Map 객체만을 사용하여 해당 점수들을 관리하여 앨리스의 점수와 비교하여 순위를 구하려고 했지만 생각보다 소스코드가 길어지고 만약에 리더보드의 등록된 플레이어의 점수와 앨리스의 점수가 더 많아진다면 시간복잡도가 O(n^2)가 나오기 때문에 데이터가 많아질 수록 최악의 시간이 나올 수 밖에 없습니다.

그러다가 블로그에서 이진탐색트리의 형태로 데이터를 저장하는 Collection 중 하나인 TreeSet 객체를 이용하여 해당 랭킹점수와 순위를 관리하기로 하였습니다.
해당 객체는 레드-블랙 트리로 구현되어 있습니다.

먼저 리더보드에 등록되어 있는 플레이어의 랭킹 점수와 순위를 다음과 같은 방식으로 저장합니다.

  • 이진트리에 중복없이 저장
  • 오름차순으로 정렬
  • Map을 이용해 점수 순위를 저장
  • 이진트리를 탐색하여 앨리스의 순위를 저장

가장 먼저 TreeSet객체를 생성합니다. 해당 객체에 플레이어들의 점수들이 저장되어 있는
배열 참조변수인 scores의 값들을 treeSet에 넣어주면 들어온 데이터의 순서와 상관없이 중복제거된 데이터들이 오름차순으로 정렬됩니다.

만약 위의 scores 배열에 100, 100 , 50, 50, 40, 40, 20, 10 정수형 타입의 값이 들어있다면 TreeSet 객체에 해당 데이터들을 넣으면 중복없이 오름차순으로 정렬된 10, 20, 40, 50, 100으로 해당 데이터들 가지게 됩니다.

결론은 앨리스의 점수의 순위를 구하는 것이고, 리더보드에 등록되어 있는 순위는 점수에 따라서 달라지기 때문에 점수가 같으면 순위도 같기 때문에 점수중복을 제거한 상태에서 순위를 관리하기 위해서 TreeSet 객체를 사용하였습니다.

HashSet 객체를 사용하지 않고 TreeSet 객체를 사용한 이유는 NavigableSet 객체 때문인데요. NavigableSet은 자바6에서 추가된 SortedSet을 확장한 인터페이스입니다.

SortedSet 클래스는 List의 하위타입이 아니므로 인덱스를 이용한 접근이 불가능하고, 궁여지책으로 역순으로 정렬을 해주는 새로운 SortedSet을 생성하곤 했습니다.
SortedSet의 이러한 단점을 보완하기 위해서, NavigableSet은 역방향 iterator를 반환하는 decendingIterator 메소드를 제공해줍니다.

자바에서 제공하는 NavigableSet 인터페이스의 대표적인 구현 클래스는 TreeSet 입니다.
TreeSet 객체가 가지고 있는 메소드 중에서 decendingSet() 메소드는 역순으로 데이터를 정렬하는 Set 객체를 반환하는 메소드 입니다.

반환된 Set의 레퍼런스를 NavigableSet 타입의 decendingSet 객체에 리턴을 해줍니다.
그러면 해당 decendingSet 객체는 100, 50, 40, 20, 10으로 역으로 정렬된 set객체를 가지게 됩니다.

그리고 HashMap 객체를 생성하여 NavigableSet이 제공해주는 메소드로 iterator 객체를 이용하여 역순으로 정렬된 각각의 점수들을 순회하여 map 객체에서 점수와 순위를 관리하게 됩니다.

이제 리더보드에 등록된 랭크 점수와 순위를 관리하는 HashMap 객체가 만들어졌기 때문에 앨리스의 점수와 비교하여 앨리스의 순위를 구하는 코드를 뽑기만 하면 되겠습니다. 이 과정도 역시 마찬가지로 NavigableSet 객체에서 제공해주는 메소드를 이용하여 간결하게 구현할 수 있었습니다.

먼저 앨리스의 플레이 횟수에 따른 점수들을 가진 alice 배열 크기만큼 순회를 합니다.
앨리스가 4번을 플레이 했다면 총 4개의 점수들이 들어가있습니다.

앨리스의 첫번째 점수를 decendingSet 객체의 first() 메소드로 decendingSet 객체의 첫번째에 저장되어 있는 값 100이랑 비교를 하게 됩니다. 만약 decendingSet 객체의 첫번째 값보다 크다면 리더보드에서 점수가 가장 높기 때문에 순위가 1인 값을 저장하면 됩니다.

만약에 리더보드에 등록되어 있는 가장 낮은 점수랑 비교하고 싶을때는 decendingSet이 제공하는 last() 메소드를 이용하여 비교를 할수 있는데요. 앨리스의 점수가 decendingSet 객체가 가지고 있는 가장 적은 데이터보다 값이 적다면
map.get을 이용하여 리더보드에 등록된 가장낮은 점수에 해당하는 순위 + 1의 값을 반환합니다.

위의 예제에서 10점이 가장 낮고 엘리스의 점수가 5점이기 때문에 map에서는 점수 10이 순위가 5로 맵핑되어 있기 때문에 저 코드에서는 앨리스의 순위르 저장하는 ranks 배열에 6값이 들어가게 됩니다.

마지막으로 엘리스의 점수가 리더보드에 등록된 랭크점수와 같다면 decendingSet의 ceiling 메소드를 사용하는데 이 메소드는 해당 파라미터로 들어온 엘리먼트가 decendingSet 객체안에 존재한다면 그 값을 리턴하고 없을시에는 해당 엘리먼트 보다 작으면서 근접한 인덱스의 엘리먼트를 리턴하게 됩니다.

만약 decendingSet.ceiling(90)을 호출하게 되면 decendingSet 객체에는 해당 엘리먼트가 존재하지 않기 때문에 90보다 작으면서 가까운 인덱스의 엘리먼트 50을 리턴하게 됩니다. 점수 50은 Map 객체에서 순위 2와 맵핑되어 있기 때문에 엘리스의 점수가 90점일 경우 map.get(decendingSet.ceiling() 메소드에서 점수 50의 순위를 리턴하면 됩니다.

주의할점은 TreeSet의 celing() 메소드와 결과 값이 다릅니다. 정렬 순서가 다르기 때문인데요. 실제로 treeSet.ceiling() 메소드를 호출하면 decendingSet.ceiling()과는 다른 값이 나오는것을 확인 할 수 있습니다.
treeSet.ceiling(90)은 100을 리턴하고, decendingSet.ceiling(90)은 50을 리턴합니다.
인덱스를 기준으로 해당 파라미터로 들어온 엘리먼트의 인덱스 보다 큰 인덱스의 엘리먼트를 리턴하기 때문입니다.
